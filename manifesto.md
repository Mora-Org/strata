# Manifesto Strata

## I. O Problema que Não Paramos para Nomear

A IA nos tornou produtivos de um jeito que nos envergonha um pouco.

Geramos mais código do que nunca. Entendemos menos do que nunca. O output cresce, a compreensão encolhe, e em algum ponto entre os dois deixamos de ser programadores para nos tornar revisores de código que não escrevemos.

Isso não é avanço. É uma forma de erosão muito bem disfarçada de produtividade.

## II. A Distinção que Importa

Existe uma diferença entre **saber que algo funciona** e **entender por que funciona**.

A maioria das ferramentas de IA atua no primeiro. Ela entrega o resultado, o código roda, o problema some. O que não some é a lacuna — a distância entre o que você produziu e o que você genuinamente compreende.

Strata trabalha no segundo. Não porque seja moralmente superior. Porque entendimento real é o único ativo que não deprecia, não é invalidado pela próxima versão de modelo, e não pode ser replicado por outra ferramenta.

## III. O Que É Uma Camada

Em geologia, cada estrato é um registro. Você pode ler a história de milhões de anos em cortes de rocha porque cada época deixou sua marca — e a marca permanece.

Entendimento funciona assim. Não é linear, não é instantâneo, e não pode ser copiado de fora para dentro. Ele se estratifica: conceito sobre conceito, prática sobre teoria, falha sobre sucesso. A camada mais recente só tem sustentação porque as anteriores existem.

Strata existe para ajudar a construir esse registro. Cada sessão é uma camada. Cada nota no vault é evidência de que a camada foi depositada por você — não importada de fora.

## IV. O Modo Vereda

Uma vereda não é uma estrada. É um caminho estreito aberto pelo esforço de quem passa — não construído para você, mas revelado pelo percurso.

No Modo Vereda, Strata não escreve seu código. Ele mostra como o terreno está organizado, aponta onde está o recurso primário, explica o mecanismo interno de um conceito, e deposita uma nota no seu vault com o que foi discutido.

O resto é seu.

Isso não é uma limitação técnica. É uma posição filosófica. Existe código que você só entende depois de ter travado nele por duas horas. Esse entendimento não pode ser comprado, só conquistado.

## V. O Modo Mestre

O Modo Mestre existe porque há momentos em que a execução importa mais que a aprendizagem.

Protótipos. Scaffolding. Partes do sistema que são instrumentais — necessárias para que algo maior funcione, mas não o objeto do seu aprendizado agora.

Ele está lá. Não é escondido nem punido. Mas é opt-in por razão: a fricção de ativá-lo deliberadamente é o lembrete de que você está fazendo uma escolha. Não um atalho inconsciente — uma decisão.

## VI. Offline como Postura

Strata roda local. Não como diferencial técnico — como postura.

Offline é foco. É ausência de notificação, de dependência, de latência que justifica distração. É a condição em que o pensamento mais difícil acontece.

Os modelos locais são menos capazes que os melhores modelos cloud. Isso também é intencional. Quando a ferramenta é mais fraca, você pensa mais. Quando você pensa mais, estratifica mais.

## VII. O Vault como Evidência

O Obsidian não é uma conveniência. É o registro geológico do seu aprendizado.

Cada nota que o Strata deposita no seu vault é uma camada com data, nível Bloom, referências primárias e backlinks para o que veio antes. Com o tempo, o grafo do vault é evidência de que o entendimento foi construído por você — não coletado de outro lugar.

Você pode defender cada nó desse grafo. Isso é o que importa.

## VIII. Para Quem é Strata

Para quem sente que produz mais e entende menos.

Para quem quer construir software que é genuinamente seu — não no sentido de autoria, mas no sentido de **compreensão**.

Para quem aceita que o caminho mais longo é o único que chega onde importa.
